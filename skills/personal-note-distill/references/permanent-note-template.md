# Permanent Note Template

Use this template for proposed notes under `personal-notes/permanent/`. Do not create or update substantive permanent-note content until the user approves the exact proposed change. Link/frontmatter formatting can be applied directly.

## Path Choices

- `personal-notes/permanent/advice/`: concrete guidance for recurring situations.
- `personal-notes/permanent/habits/`: repeated behaviors to practice.
- `personal-notes/permanent/principles/`: high-level philosophy and decision heuristics.

## Frontmatter

```yaml
---
type: permanent-note
created: YYYY-MM-DD
updated: YYYY-MM-DD
derived_from: []
tags: []
---
```

## Examples of Good Advice Shape

- "When I leave an advisor meeting, write the unresolved decisions first, then todos. Otherwise action items crowd out the reasoning that made them important."
- "When a project idea has no linked experiment after a week, either design a small test or demote the idea. Unlinked ideas create false progress."
- "When a paper changes my view, update both the paper survey and the related permanent principle. Otherwise my future behavior keeps following the old belief."
